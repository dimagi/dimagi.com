{% blocktrans %} Test your solution {% endblocktrans %}

{% blocktrans %} While the typical QA process is crucial to ensure the app you develop functions as you expect, user acceptance testing (or UAT) validates whether your expectations will actually solve the problems your users face. {% endblocktrans %}

{% blocktrans %} Receiving consistent feedback throughout the app building process (in both the design and build phases) is vital to developing a solution that truly solves the problems you are trying to tackle and helps you avoid any surprises when you test the final product. {% endblocktrans %}

{% blocktrans %} Once your first version is complete, but before launch, you should test the system in its entirety with real users in the setting in which the tool will be used. This is user acceptance testing. {% endblocktrans %}

{% blocktrans %} From the hundreds of applications we have helped develop, we have a number of tips to share around how to run a quality and insightful UAT process: {% endblocktrans %}

{% include "svg/pillar/mdc/chapter6/icon5.html" %}
{# .column #}

{% blocktrans %} Identify the right users. {% endblocktrans %} {% blocktrans %} Make sure you have a representative sample of your users to ensure that everyone who ends up using the app is accounted for.
a) High & low performing
b) High & low digital literacy
c) Rural and urban {% endblocktrans %}

{# .column #}
{# .grid #}
{% include "svg/pillar/mdc/chapter6/icon6.html" %}
{# .column #}

{% blocktrans %} Ask open-ended, rather than leading, yes/no questions. {% endblocktrans %} {% blocktrans %} Just like the questions you ask in the app, the goal of user testing is to uncover clear, unbiased feedback on how you might improve your platform. {% endblocktrans %}

{# .column #}
{# .grid #}
{% include "svg/pillar/mdc/chapter6/icon7.html" %}
{# .column #}

{% blocktrans %} Observe, don't demo. {% endblocktrans %} {% blocktrans %} You want to see the users interact with the system in as realistic an environment as possible, which means you are not there to hand-hold. See what flows they get caught up on and which go smoothly. This will be representative of when the users you might never meet get their hands on your app. {% endblocktrans %}

{# .column #}
{# .grid #}
{% include "svg/pillar/mdc/chapter6/icon8.html" %}
{# .column #}

{% blocktrans %} Consolidate observations and feedback. {% endblocktrans %} {% blocktrans %} Organizing your feedback in a structured way helps to get a clear picture of the trends. It also ensures that you can easily compare and contrast the feedback with that on future iterations of the app. {% endblocktrans %}

{# .column #}
{# .grid #}
{% include "svg/pillar/mdc/chapter6/icon9.html" %}
{# .column #}

{% blocktrans %} Sort through the noise. {% endblocktrans %} {% blocktrans %} Develop a clear idea of your priorities and how the feedback you receive aligns with them. You cannot incorporate every change request, so have a decision-making process that all stakeholders agree on, and stick to it. {% endblocktrans %}

{# .column #}
{# .grid #}
{% include "svg/pillar/mdc/chapter6/icon10.html" %}
{# .column #}

{% blocktrans %} Factor enough time in your project plan to incorporate feedback, {% endblocktrans %} {% blocktrans %} and QA your app thoroughly again before deploying. {% endblocktrans %}

{# .column #}
{# .grid #}

{% blocktrans %} Along with testing the effectiveness of the system you built in the eyes of your users, UAT helps test how the technology fits into your overall project objectives. The user testing phase is a great opportunity to review everything from your results framework and information flow diagram (or data collection plan outline) to your data requirements and user stories to ensure that your new mobile data collection tool delivers on everything you expected from it. {% endblocktrans %}
