{% blocktrans %} Build your mobile data collection solution {% endblocktrans %}

{% blocktrans %} Now, the fun begins! It’s time to dive into the mobile data collection solution you selected and build the mobile application that your team will use in the field. {% endblocktrans %}

{% blocktrans %} Build your outline and fill it in {% endblocktrans %}

{% blocktrans %} Time to build! With the overall structure and workflow that you designed and validated in the Design phase, start integrating the more detailed content and logic into your forms. What are the questions you wrote when you established your data collection standards? Add them in! Are there any documents, videos, or messages your app will need to share at certain times? Put those in, too! {% endblocktrans %}

{% blocktrans %} If you are using CommCare, our form builder offers additional features beyond basics (e.g.. skip logic/display conditions and validation conditions), which you might consider adding to your app for improved user experience: {% endblocktrans %}

{% blocktrans %} In a module case list {% endblocktrans %}

{% blocktrans %} Registration from the case list {% endblocktrans %}

{% blocktrans %} Make the registration form directly accessible from a case list, so the user can go from searching for a patient to registering a new one seamlessly. {% endblocktrans %}

{% blocktrans %} "Search Only" properties in the case list {% endblocktrans %}

{% blocktrans %} Even if you only display a few properties on your case list, you can still search by others (e.g. District) that will be important to your program. "Search Only" properties give the user more search and filtering options without overcrowding the phone screen. {% endblocktrans %}

{% blocktrans %} Icons in the case list {% endblocktrans %}

{% blocktrans %} Use conditional icons in the case list to visually highlight important details about each case. {% endblocktrans %}

{% blocktrans %} In your forms {% endblocktrans %}

{% blocktrans %} Calculations in hidden values {% endblocktrans %}

{% blocktrans %} Use hidden values (a special question type) in your form to carry out calculations that are not visible to the user. For example, calculate a patient’s age in years and months based on the date of birth question asked during patient registration, or calculate the next visit date based on today's date. {% endblocktrans %}

{% blocktrans %} Define default values {% endblocktrans %}

{% blocktrans %} Pre-load an answer to a question in your form by loading a case property, calculation, or text into the default value field. This helps the user out by suggesting a response but allowing them to edit it in the form. {% endblocktrans %}

{% blocktrans %} Working with multiple app builders {% endblocktrans %}

{% blocktrans %} Sometimes, programs are fortunate enough to have multiple people working on their app. Unfortunately, this can also lead to confusion and versioning issues. Our team has a few suggestions when it comes to working with multiple app builders: {% endblocktrans %}

{% include "svg/pillar/chapter6/icon1.html" %}
{# .column #}

{% blocktrans %} Define clear workstream owners and divide up the app building accordingly. {% endblocktrans %} {% blocktrans %} Most tools do not allow multiple builders at once, so be clear about who will be working in the platform and when. {% endblocktrans %}

{# .column #}
{# .grid #}
{% include "svg/pillar/chapter6/icon2.html" %}
{# .column #}

{% blocktrans %} Make and document builds as frequently as possible. {% endblocktrans %} {% blocktrans %} Quick, but incremental updates will make it easier to spot and solve any bugs that crop up. Check to see if your platform allows you to control versions to see who made what updates and when. {% endblocktrans %}

{# .column #}
{# .grid #}
{% include "svg/pillar/chapter6/icon3.html" %}
{# .column #}

{% blocktrans %} Set clear communication mechanisms with your team. {% endblocktrans %} {% blocktrans %} Daily or every-other-day stand-up meetings for quick check-ins are helpful. {% endblocktrans %}

{# .column #}
{# .grid #}
{% include "svg/pillar/chapter6/icon4.html" %}
{# .column #}

{% blocktrans %} Build in consistent feedback mechanisms from your users. {% endblocktrans %} {% blocktrans %} This will help keep a centralized database of feedback so that all app builders are aware of their users' frustrations and suggestions. This is especially important when your users and app development team are in different locations and those conversations cannot happen face to face. {% endblocktrans %}
